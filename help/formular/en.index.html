General information

  The "Advanced Form" extension makes it possible to create forms and landing pages that are optimised for tablet and mobile displays and comply with the Web Content Accessibility Guidelines (WCAG).

The creation of advanced forms is done by a multi-part wizard:

  • A "Design" section allows you to build the form from blocks of structure, content and input fields.
  • A "Settings" section gives access to the properties of the form such as its language, the validation message, publication dates and pre/post publication messages as well as the rights to view/modify the form by other users of the Eudonet database.
  • A "Publication" section brings together the form's publication button, the public sharing link and the code to be copied to integrate the form into an existing site.

Remark

To benefit from all the form creation functionalities, we invite you to use the Chrome, Firefox or Safari browsers.

Design

The "Design" tab allows you to define the content of your form by dropping blocks of structure, content and input fields.


The Blocks to drop are available in the right menu under the following tab :

Structural blocks

  "Canvas" block This block is mandatory on the form, it is the basic element of the page on which the other elements will be deposited.
"1 Column" block This block allows you to add a column that takes up the entire available width where the block is placed.
"2 Columns" block This block allows you to add 2 columns each taking 50% of the available width where the block is placed. The 2 columns pass one under the other on the tablets in portrait mode and cell phones or when the width of the window is reduced.
"2 Columns 30%/70%" block This block allows to add 2 columns taking respectively 30% and 70% of the available width where the block is placed. The 2 columns pass one under the other on the tablets in portrait mode and cell phones or when the width of the window is reduced.
"2 Columns 70%/30%" block This block allows to add 2 columns taking respectively 70% and 30% of the available width where the block is placed. The 2 columns pass one under the other on the tablets in portrait mode and cell phones or when the width of the window is reduced.
"3 Columns" block This block allows you to add 3 columns, each taking up 1/3 of the available width where the block is placed. The 3 columns pass underneath each other on tablets in portrait mode and mobile phones or when the width of the window is reduced.

Remark

Each of the structure blocks has its own dimensions, margins and background colour.

Modify the dimensions, margins and background colour of a block from the right menu

By selecting an element of the graphic part, the right menu will automatically be placed on the "Brush" tab allowing access to the properties of the element:

  • The properties related to the background colour and the colour, shape and thickness of the borders are combined in the chapter "Colour and border".
  • The properties related to dimensions, margins and padding can be found in the chapter "Dimensions and margins".

Content blocks

The "Text" block

This block is used to add a text area to the form. A toolbar allows you to bold, italic, underline or strike through the text, add a link or insert merge fields :



Text properties such as alignment, colour, font and size can be changed in the Alignment and Typography chapters of the right-hand menu.


As with the other elements, it is also possible to change the dimensions and margins of the image block by opening the corresponding menus in the right-hand menu (see chapterModify the dimensions, margins and background colour of a block from the right menu).

Add a link

To add a link, in a text frame, select the text that will carry the link and click on the link icon in the toolbar.


Then click once outside the text area and then click on the text that has just been turned into a link (it should be underlined now), you will then see the properties of the link appear in the right-hand menu.


Attention It is important to start your Href (link URL) with "https://" or "http://" for the link to open correctly.
Feel free to check your links after publishing your form and before sharing it.

Tip for pasting in plain text

To avoid using the source formatting when copying and pasting, the "Chrome" browser offers a "Paste as plain text" function in the right-click menu.

Alternative text for an image

In order to ensure RGAA compliance by the form, it is important to define the replacement text for the image, it will be used when the image cannot be displayed or if the internet user uses a screen reader in the case of a visual disability for example. This text is defined in the "item parameters" which are displayed in the right-hand menu when you click on the Image block:

Changing the alignment of an image

If your image is not the full width of the area and you wish to centre or right-align it, select the parent item and from the "Alignment" menu in the right-hand menu, select the alignment that suits you.

The "Image" block

This block allows you to add an image to the form, by default it does not exceed the width available in the area where it has been dropped and its width automatically reduces to fit the size of the screen displaying the form.
As for the other elements, it is also possible to modify the dimensions and margins of the Image block by clicking on the "Brush" pictogram in the right-hand menu (see chapter Modify the dimensions, margins and background colour of a block from the right menu).

Input blocks

The "Form" block

This block allows you to add a set of elements including a centred area 800px wide by default and a validation button at the bottom of this area. It is in this Form block that the input blocks can be added.

Change the background colour of the form block

If you wish to change the background colour of the form block, you need to select the element in the centre of the block which includes a free area and the button.

To select this item, you can click on one of the items in the form block and scroll up using the up arrow on the toolbar below :

Then in the chapter "Colour and border" in the right-hand menu, you can change the "Background colour" of the block.

Change the width of the form block

If you wish to enlarge or reduce the width taken by the form block, you need to select the element in the centre of the block which includes a free area and the button.

To select this item, you can click on one of the items in the form block and scroll up using the up arrow on the toolbar below :

Then in the chapter "Size and margin" of the right menu, the default "Max width" of 800 px is visible. You can then change it to a new maximum width in pixel or %.

"Short Text" block This block is used to add a short text input field, it allows the user to enter, for example, his surname, first name or a text on a line.
"E-mail" block block allows you to add an email input field, when the internet user enters it the format of the email address entered is checked.
"Telephone" block This block allows you to add an input field for a phone, there is no specific control for the phone format.
"Check box" block This block allows you to add a checkbox in the form.
To access the properties of this block, click on the checkbox.
"Numeric" block This block is used to add a numeric input field to the form.
"Date" block This block is used to add a date entry field to the form. The "Time" property is used to define whether the time is also to be entered by the user.
"Long text" block This block is used to add a long text input field to the form.
The "Number of lines" property is used to define the height of the input zone on the form. The input is limited to 4000 characters.
"Dropdown list" block This block is used to add an entry field from a single-choice drop-down list.
"Multiple choice" block This block is used to add an entry field from a multiple-choice drop-down list.

Changing the wording and properties of an input block

All the types of input blocks listed below work on the same principle, when a block is dropped, it is composed of 2 elements: a label and an input field.
By clicking on the label, you can modify it.

By clicking on the input field, you can access its properties in the right-hand menu.

Properties vary depending on the type of input block but some are common:

  • Watermark text : Help text appearing in the input field to tell the user what to expect in the field. The watermark text disappears as soon as the user starts typing in the field. If no watermark is set here then the watermark set in Eudonet for the associated field (see below) applies.
  • Associated field : Eudonet field in which the data entered by the Internet user when validating the form is recorded. If the form is used to complete/update an existing form, then the field is pre-filled with the data in Eudonet when the form is opened. This is the case, for example, for a form sent by email to recipients in the Eudonet database.
  • Make this field mandatory : Checking this box makes the entry of this field mandatory for the validation of the form.

The properties available may vary depending on the type of block:

  • Check box : There is no watermark, but you can set whether the box is checked by default.
  • Date : An option allows you to define if the time is also to be entered by the Internet user.
  • Long text : A property allows you to define the height of the input field in number of lines.

Settings

The "Settings" tab gives access to the properties of the form, such as its language, the validation message, publication dates and pre/post publication messages, the rights to view/modify the form by other users of the Eudonet database and the personalisation options.

General settings

Form language

It is set by default to the user's language, it can be changed to change the language in which the form control messages will be displayed, such as those for mandatory fields not filled in or those for incorrect email address entry.

Thank you message

When the form is validated by the Internet user, it is possible to display a thank you message or to redirect it to another Internet URL. In the case of the message, it is possible to layout it and insert merge fields containing information that the user has just entered.

Scheduling

Start date

This date is used to define the date on which the content of the form will be accessible and the message to be displayed if an Internet user accesses it before this date.

End date

This date makes it possible to define the date on which the content of the form will no longer be accessible and the message to be displayed if an Internet user accesses it after this date.

Remark

Start and end dates and associated messages are only taken into account if the form is published.

Sharing on social networks

The sharing information set up in the "Publication" tab, such as the image, title and description, is taken into account even if the form is shared before being accessible.

Security

These rights are used to define which Eudonet users have access to the form for viewing and/or editing.

Remark

Access rights are not transferred when a form is duplicated.

Personalisation

This option allows you to set the default colour of the links in the form.

These options allow you to set the accent colour and the colour of the value of the input fields.
The accent colour is the colour of the button and fields as they are entered by the user.

These options allow you to set the size and font of the input fields.

Remark

The personalisation is visible on the published form but is not taken into account in the "Design" step.

Publication

The "Publication" tab brings together the button for publishing the form, the public sharing link, the sharing links on social networks and the code to be copied to integrate the form into an existing site.

Publish and share

Publish

By activating the switch, the form is changed to "Published" status and becomes accessible according to the scheduling defined in the settings.

Remark

On publication, the form is saved and all input fields are checked to ensure that they all have an associated field.

Public link

This link is available after publishing the form and allows the form to be shared publicly.

Social networks

The 3 buttons allow you to share the form on Facebook, LinkedIn or Twitter. The image, title and description that will appear in the publication are customisable.

Save the latest changes before sharing

After you have made your final changes to the sharing information, please remember to save your changes so that they will be taken into account when you share your form.

Format and dimensions of the image

The image must be in png, svg, jpeg or gif format and must not exceed 5 MB.

In order to optimise the display of your publication it is advisable to use:

  • For Facebook, an image of 1200x630 px and at least 200x200 px
  • For LinkedIn, an image of 1200x627 px
  • For Twitter, an image of at least 600x335 px

Maximum size for title and description

  • The title is limited to 70 characters
  • The description is limited to 155 characters

Embedding the form

Entire form

As soon as the form is published, the code to integrate the form in a page of a website is available in this menu. To integrate the form, copy the code and insert it in the source code of the web page where you want the form to appear. The embedded form corresponds to the public form, so there is a Google V2 reCAPTCHA that is automatically added above the validation button.
